\begin{itemize}
    \item \textbf{Web Application (Frontend/Backend):} \\
    \url{https://github.com/bambiiisadeer/TempTrack.git}
    \item \textbf{IoT Devices (Microcontroller \& Sensors):} \\
    \url{https://github.com/phuennae/TempTrack_IoTs.git}
\end{itemize}

\section{ขั้นตอนการติดตั้งเพื่อใช้งานระบบ}
ในการติดตั้งและเตรียมความพร้อมของระบบโครงงานบน Cloud Server (AWS EC2 - Ubuntu) ได้แบ่งขั้นตอนการดำเนินงานตามองค์ประกอบหลักของระบบออกเป็น 5 ส่วน ดังนี้

\subsection{การติดตั้งและรันระบบเว็บแอปพลิเคชัน}
เว็บแอปพลิเคชันส่วนหน้า (Frontend) ถูกพัฒนาด้วยเฟรมเวิร์ก Node.js เพื่อเชื่อมต่อและแสดงผลข้อมูลจากบล็อกเชน โดยมีขั้นตอนการติดตั้งดังนี้:

\begin{verbatim}
# 1. เข้าสู่โฟลเดอร์ของ Web Application
cd TempTrack

# 2. ลง Dependencies ของระบบ
npm install

# 3. รันระบบในโหมด Production
npm run build
npm start
\end{verbatim}
\textit{*หมายเหตุ: หากต้องการเข้าชมผ่านเบราว์เซอร์ ต้องทำการเปิดพอร์ต 3000 (หรือพอร์ตที่ใช้งาน) ในหน้า Security Group ของ AWS EC2}

\vspace{2em}

\subsection{การติดตั้งและตั้งค่า MQTT Broker บน AWS EC2}
ใช้โปรโตคอล MQTT สำหรับรับส่งข้อมูลอุณหภูมิจากอุปกรณ์ IoT โดยทำการติดตั้ง Mosquitto Broker บนเซิร์ฟเวอร์ตามขั้นตอนดังนี้:

\begin{verbatim}
# 1. อัปเดตรายการแพ็กเกจ
sudo apt-get update

# 2. ลง Mosquitto Broker และ Client
sudo apt-get install mosquitto mosquitto-clients -y

# 3. ตั้งค่าให้ทำงานโดยอัตโนมัติ
sudo systemctl enable mosquitto
sudo systemctl start mosquitto
\end{verbatim}
\textit{*หมายเหตุ: จำเป็นต้องเปิดพอร์ต 1883 ในหน้า Security Group ของ AWS EC2 เพื่ออนุญาตให้ส่งข้อมูลเข้าสู่ระบบได้}

\vspace{2em}

\subsection{การติดตั้งและตั้งค่าเครือข่าย Hyperledger Fabric บน AWS EC2}
การรันเครือข่ายบล็อกเชนจำเป็นต้องมีการติดตั้ง Docker และเตรียมระบบพื้นฐาน ดังนี้:

\begin{verbatim}
# 1. ลงเครื่องมือพื้นฐานและ Docker Engine
sudo apt-get install curl git jq docker.io docker-compose -y
sudo usermod -aG docker $USER

# 2. ดึงสคริปต์และลงไบนารีของ Hyperledger Fabric
curl -sSLO https://raw.githubusercontent.com/hyperledger/fabric/main/scripts/install-fabric.sh
chmod +x install-fabric.sh
./install-fabric.sh docker samples binary
\end{verbatim}

\vspace{2em}

\subsection{การเตรียมอุปกรณ์ไอโอทีและการอัปโหลดโค้ด}
ก่อนเริ่มการส่งข้อมูลเข้าสู่ระบบ ต้องทำการตั้งค่าพารามิเตอร์ภายในโค้ดและอัปโหลดลงสู่บอร์ดไมโครคอนโทรลเลอร์ ดังนี้:

\begin{itemize}
    \item \textbf{WiFi Configuration:} แก้ไขค่าตัวแปร \texttt{ssid} และ \texttt{password} ให้ตรงกับเครือข่ายไร้สายในพื้นที่
    \item \textbf{Server Configuration:} ระบุหมายเลข IP Address (Public IP) ของ AWS EC2 Server ในส่วนของ MQTT Broker Host
    \item \textbf{Code Uploading:} ใช้โปรแกรม Arduino IDE ในการอัปโหลดโค้ด (Burn) ลงสู่บอร์ดเพื่อให้เริ่มทำงาน
\end{itemize}

\vspace{2em}

\subsection{การรันเซอร์วิสเชื่อมต่อข้อมูล IoT เข้าสู่บล็อกเชน}

\chapter{โครงสร้างซอร์สโค้ด}
\section{โครงสร้างซอร์สโค้ดฝั่ง IoT}
หน้าที่ของไฟล์หลักในส่วนของอุปกรณ์มีรายละเอียดดังนี้:
\begin{itemize}
    \item[] \textbf{\texttt{DS18B20\_example.ino}} : โค้ดสำหรับทดสอบการทำงานของเซ็นเซอร์เบื้องต้น
    \item[] \textbf{\texttt{Temptrack\_MQTT.ino}} : โค้ดหลักที่ใช้ในการประมวลผลและส่งข้อมูลผ่าน MQTT
\end{itemize}

\section{ซอร์สโค้ดฝั่ง Web Application และระบบเชื่อมต่อ}
โครงสร้างไฟล์ภายในระบบเว็บแอปพลิเคชันและตัวกลางเชื่อมต่อข้อมูล มีรายละเอียดดังนี้:
\begin{itemize}
    \item[] \textbf{\texttt{server.js}} : ไฟล์หลักที่จัดการการเรียกใช้ Smart Contract เพื่อบันทึกข้อมูลลงบล็อกเชน
    \item[] \textbf{\texttt{/public}} : เก็บไฟล์ Static สำหรับการแสดงผลบนเว็บหน้าบ้าน (Frontend)
    \item[] \textbf{\texttt{/chaincode}} : ซอร์สโค้ดของ Smart Contract (Chaincode)
    \item[] \textbf{\texttt{package.json}} : ไฟล์ระบุรายการ Dependencies และคำสั่งสำหรับการรันระบบ
\end{itemize}

ตัวอย่างลอจิกสำคัญในการดักฟังข้อมูลและบันทึกลงบล็อกเชนในไฟล์ \texttt{server.js}:
\begin{verbatim}
client.on('message', async (topic, message) => {
    const data = JSON.parse(message.toString());
    console.log(`Recv: ${data.sensor_id} | ${data.temp}`);

    try {
        await contract.submitTransaction('CreateRecord', data.temp);
        console.log('Saved to Blockchain!');
    } catch (e) {
        console.log('Error: ', e);
    }
});
\end{verbatim}
