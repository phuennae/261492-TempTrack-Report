\section{โครงสร้างของระบบ }


\subsection{การออกแบบระบบพื้นฐานของโครงงาน }

  ระบบการจัดการซัพพลายเชนสำหรับการขนส่งสินค้าแช่แข็งที่พัฒนาขึ้นนี้มีวัตถุประสงค์เพื่อเพิ่มความโปร่งใสและความน่าเชื่อถือของข้อมูลเกี่ยวกับสภาพแวดล้อมของสินค้าในระหว่างการขนส่ง โดยการใช้เทคโนโลยีบล็อกเชน และ อินเทอร์เน็ตทุกสรรพสิ่งหรือไอโอที เพื่อให้สามารถตรวจสอบข้อมูลอุณหภูมิแบบเรียลไทม์ได้อย่างแม่นยำ ระบบนี้มีการออกแบบโครงสร้างพื้นฐานโดยแบ่งเป็นหลายองค์ประกอบหลัก ได้แก่: 

\begin{enumerate}
    \item บันทึกข้อมูลสินค้า
    \begin{itemize}
        \item เจ้าหน้าที่ทำการบันทึกรายละเอียดของสินค้า รวมถึงข้อมูลที่อยู่และช่วงอุณหภูมิที่เหมาะสมสำหรับสินค้าดังกล่าว 
        \item ใช้อุปกรณ์อินเทอร์เน็ตทุกสรรพสิ่งหรือไอโอที เช่น เซ็นเซอร์วัดอุณหภูมิที่ติดตั้งในภาชนะบรรจุสินค้าเพื่อบันทึกข้อมูลอุณหภูมิที่เกิดขึ้นจริงในระหว่างการขนส่ง 
    \end{itemize}
    \item ระบบฐานข้อมูล
    \begin{itemize}
        \item ใช้โปรแกรมDocker ในการจัดการระบบบริการต่างๆ เพื่อให้มั่นใจว่าการทำงานของระบบสามารถปรับขนาดและปรับใช้ได้อย่างสะดวก 
        \item ใช้การติดตามอัจฉริยะภายในบล็อกเชน เพื่อบันทึกและยืนยันข้อมูลที่ได้รับจากเซ็นเซอร์โดยอัตโนมัติ ป้องกันการเปลี่ยนแปลงหรือปลอมแปลงข้อมูล
    \end{itemize}
    \item การประมวลผลและจัดเก็บข้อมูล 
    \begin{itemize}
        \item เมื่อเซ็นเซอร์ตรวจวัดอุณหภูมิ ข้อมูลที่ได้รับจะถูกเข้ารหัสและส่งไปยังระบบจัดเก็บข้อมูลที่อยู่บนบล็อกเชน
        \item ฐานข้อมูลภายในระบบถูกออกแบบมาเพื่อเก็บข้อมูลที่สำคัญ เช่น ข้อมูลอุณหภูมิ เวลาที่บันทึก สถานะสินค้า และข้อมูลเกี่ยวกับการขนส่ง 
    \end{itemize}
    \item ส่วนติดต่อผู้ใช้สำหรับการตรวจสอบและวิเคราะห์ข้อมูล
    \begin{itemize}
        \item เจ้าหน้าที่สามารถเข้าถึงข้อมูลที่บันทึกไว้ผ่านอินเทอร์เฟซบนแพลตฟอร์มเว็บ เพื่อวิเคราะห์และตรวจสอบความถูกต้องของข้อมูล 
        \item ระบบมีฟังก์ชันแสดงรายงานและแจ้งเตือนในกรณีที่อุณหภูมิของสินค้าผิดปกติจากช่วงที่กำหนด 
    \end{itemize}
    \item การบูรณาการบล็อกเชน
    \begin{itemize}
        \item บล็อกเชนถูกใช้เป็นระบบจัดเก็บข้อมูลหลักเพื่อให้เกิดความโปร่งใสและป้องกันกาเปลี่ยนแปลงข้อมูลที่ไม่พึงประสงค์ 
        \item ข้อมูลที่ได้รับจากเซ็นเซอร์ไอโอที จะถูกเข้ารหัสและบันทึกลงในบล็อกเชนทันทีที่มีการเปลี่ยนแปลงของข้อมูลอุณหภูมิ 
        \item การใช้การติดตามอัจฉริยะ ช่วยให้สามารถดำเนินการตรวจสอบและยืนยันข้อมูลได้โดยอัตโนมัติ .
    \end{itemize}
\end{enumerate}

\section{ขั้นตอนการดำเนินงานของระบบ }

\begin{enumerate}
    \item การบันทึกข้อมูลสินค้า
    \begin{itemize}
        \item เจ้าหน้าที่ทำการป้อนข้อมูลสินค้า เช่น ชื่อสินค้า รหัสสินค้า ที่อยู่ของต้นทางและปลายทาง และช่วงอุณหภูมิที่กำหนด
        \item ระบบจะสร้างรายการข้อมูลสำหรับการขนส่งสินค้าและเชื่อมโยงกับเซ็นเซอร์ที่เกี่ยวข้อง
    \end{itemize}
    \item การตรวจสอบและบันทึกอุณหภูมิแบบเรียลไทม์ 
    \begin{itemize}
        \item เซ็นเซอร์ที่ติดตั้งในภาชนะบรรจุสินค้าจะทำการวัดอุณหภูมิอย่างต่อเนื่อง และส่งข้อมูลไปยังระบบกลางผ่านเครือข่าย IoTs 
        \item ข้อมูลจะถูกเข้ารหัสและบันทึกลงใน บล็อกเชน เพื่อป้องกันการแก้ไขข้อมูล 
    \end{itemize}
    \item การแจ้งเตือนเมื่อพบค่าผิดปกติ 
    \begin{itemize}
        \item หากอุณหภูมิของสินค้าสูงหรือต่ำกว่าช่วงที่กำหนด ระบบจะทำการแจ้งเตือนไปยังเจ้าหน้าที่ทันทีผ่านแพลตฟอร์มเว็บ
        \item เจ้าหน้าที่สามารถดำเนินการแก้ไขปัญหาหรือแจ้งให้ผู้ที่เกี่ยวข้องทราบได้โดยตรง 
    \end{itemize}
    \item การตรวจสอบและรายงานผล 
    \begin{itemize}
        \item ข้อมูลที่ได้รับจะสามารถเข้าถึงผ่านอินเทอร์เฟซของแพลตฟอร์มเว็บเพื่อการวิเคราะห์และการสร้างรายงาน 
        \item ระบบมีฟังก์ชันสำหรับสร้างรายงานสรุปเพื่อให้ผู้บริหารสามารถนำไปใช้ในการตัดสินใจ 
    \end{itemize}
    \item การปิดกระบวนการขนส่ง 
    \begin{itemize}
        \item เมื่อสินค้าถึงปลายทาง ระบบจะบันทึกสถานะสุดท้ายและทำการตรวจสอบข้อมูลอุณหภูมิย้อนหลัง 
        \item บล็อกเชนจะบันทึกข้อมูลสุดท้ายเกี่ยวกับการขนส่ง เพื่อให้สามารถอ้างอิงได้ในกรณีที่มีข้อพิพาทหรือปัญหาเกี่ยวกับสินค้า 
    \end{itemize}
\end{enumerate}
