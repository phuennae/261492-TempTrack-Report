\section{โครงสร้างของระบบ }


\subsection{การออกแบบระบบพื้นฐานของโครงงาน }

  ระบบบริหารจัดการห่วงโซ่อุปทานสำหรับการขนส่งสินค้าควบคุมอุณหภูมินี้ ถูกพัฒนาขึ้นโดยมีวัตถุประสงค์เพื่อเพิ่มความโปร่งใสและความน่าเชื่อถือของข้อมูลสภาพแวดล้อมระหว่างการขนส่ง โดยบูรณาการเทคโนโลยีบล็อกเชน (Blockchain) ร่วมกับเทคโนโลยีอินเทอร์เน็ตของสรรพสิ่ง (Internet of Things: IoT) เพื่อให้สามารถตรวจสอบและติดตามข้อมูลอุณหภูมิแบบเรียลไทม์ได้อย่างแม่นยำ โครงสร้างพื้นฐานของระบบถูกออกแบบโดยแบ่งออกเป็น 5 องค์ประกอบหลัก ได้แก่: 

\begin{enumerate}
    \item การนำเข้าและตรวจจับข้อมูล
    \begin{itemize}
        \item ส่วนการบันทึกข้อมูลโดยเจ้าหน้าที่: เจ้าหน้าที่จะดำเนินการป้อนรายละเอียดที่เกี่ยวข้องกับการขนส่ง เช่น ข้อมูลสินค้า ที่อยู่ และการกำหนดช่วงอุณหภูมิที่เหมาะสมสำหรับสินค้านั้น ๆ 
        \item ส่วนการตรวจจับด้วยอุปกรณ์ IoT: อาศัยเซนเซอร์วัดอุณหภูมิที่ติดตั้งไว้ภายในกล่องบรรจุสินค้า เพื่อทำหน้าที่ตรวจวัดและบันทึกค่าอุณหภูมิที่เกิดขึ้นจริงอย่างต่อเนื่องตลอดกระบวนการขนส่ง 
    \end{itemize}
    \item โครงสร้างพื้นฐานและระบบจัดเก็บข้อมูล
    \begin{itemize}
        \item ใช้เทคโนโลยีคอนเทนเนอร์ในการบริหารจัดการเซอร์วิสต่าง ๆ ของระบบ เพื่อให้มั่นใจว่าการทำงานมีความเสถียร รองรับการขยายขนาด และสามารถนำไปติดตั้งใช้งานจริงได้อย่างสะดวกรวดเร็ว 
        \item มีระบบฐานข้อมูลภายในที่ถูกออกแบบมาเพื่อจัดเก็บสถานะล่าสุดของข้อมูลที่สำคัญ เช่น ข้อมูลอุณหภูมิ รอยเวลาที่บันทึก และสถานะปัจจุบันของสินค้า เพื่อการเรียกดูที่รวดเร็ว
    \end{itemize}
    \item การประมวลผลและส่งผ่านข้อมูล 
    \begin{itemize}
        \item เเมื่อเซนเซอร์ IoT ทำการตรวจวัดค่าอุณหภูมิได้ ข้อมูลดังกล่าวจะถูกส่งผ่านเครือข่ายเข้าสู่ระบบส่วนกลาง
        \item ข้อมูลที่ได้รับจะถูกนำไปประมวลผลและส่งต่อไปยังระบบจัดเก็บข้อมูลบนเครือข่ายบล็อกเชน 
    \end{itemize}
    \item ส่วนติดต่อผู้ใช้งานและการวิเคราะห์ข้อมูล
    \begin{itemize}
        \item เจ้าหน้าที่และผู้ที่เกี่ยวข้องสามารถเข้าถึงข้อมูลประวัติการขนส่งผ่านหน้าเว็บแอปพลิเคชัน เพื่อประโยชน์ในการวิเคราะห์และตรวจสอบความถูกต้องของข้อมูล 
        \item ระบบมีฟังก์ชันการแจ้งเตือนทันทีในกรณีที่ตรวจพบว่าค่าอุณหภูมิของสินค้าออกนอกช่วงเกณฑ์มาตรฐานที่กำหนดไว้ พร้อมทั้งสามารถแสดงผลสรุปในรูปแบบรายงานได้ 
    \end{itemize}
    \item การบูรณาการเทคโนโลยีบล็อกเชน
    \begin{itemize}
        \item เครือข่ายบล็อกเชนถูกนำมาใช้เป็นบัญชีธุรกรรมหลักในการจัดเก็บข้อมูล เพื่อสร้างความโปร่งใสและป้องกันการดัดแปลงแก้ไขข้อมูลอุณหภูมิโดยไม่ได้รับอนุญาต
        \item ข้อมูลที่ถูกส่งมาจากอุปกรณ์ IoT จะถูกบันทึกลงในบล็อกเชนทันทีที่มีการเปลี่ยนแปลงสถานะ 
        \item มีการประยุกต์ใช้สัญญาอัจฉริยะเข้ามาช่วยดำเนินการตรวจสอบและยืนยันความถูกต้องของข้อมูลการขนส่งลงสู่บล็อกเชนได้โดยอัตโนมัติ
    \end{itemize}
\end{enumerate}

\section{ขั้นตอนการดำเนินงานของระบบ }

\begin{enumerate}
    \item การบันทึกข้อมูลสินค้า
    \begin{itemize}
        \item เจ้าหน้าที่ดำเนินการป้อนข้อมูลที่เกี่ยวข้องกับการขนส่ง เช่น ข้อมูลผู้ส่ง ข้อมูลผู้รับ รายละเอียดของสินค้า และช่วงอุณหภูมิควบคุมที่กำหนด
        \item ระบบจะทำการสร้างรายการขนส่งและออกหมายเลขติดตามพัสดุ เพื่อใช้สำหรับตรวจสอบสถานะของสินค้าตลอดกระบวนการ
    \end{itemize}
    \item การตรวจสอบและบันทึกอุณหภูมิแบบเรียลไทม์ 
    \begin{itemize}
        \item เอุปกรณ์เซนเซอร์ IoT ที่ติดตั้งบริเวณกล่องบรรจุสินค้า จะทำการวัดค่าอุณหภูมิอย่างต่อเนื่อง และส่งข้อมูลผ่านเครือข่ายอินเทอร์เน็ตไปยัง Amazon EC2 Server
        \item ข้อมูลอุณหภูมิจะถูกส่งผ่านโปรโตคอล MQTT เพื่อนำไปแสดงผล และถูกส่งไปบันทึกลงในระบบบล็อกเชน เพื่อรับประกันความปลอดภัยและป้องกันการแก้ไขข้อมูลย้อนหลัง
        \item ผู้มีส่วนได้ส่วนเสียสามารถติดตามสถานะอุณหภูมิของสินค้าได้แบบเรียลไทม์ผ่านเว็บแอปพลิเคชัน
    \end{itemize}
    \item การแจ้งเตือนเมื่อพบค่าผิดปกติ 
    \begin{itemize}
        \item หากอุณหภูมิของสินค้าสูงหรือต่ำกว่าช่วงที่กำหนด ระบบจะแสดงการแจ้งเตือนบนแพลตฟอร์มเว็บไซต์ทันที
        \item เจ้าหน้าที่หรือผู้ควบคุมระบบสามารถรับทราบปัญหา และดำเนินการแก้ไขสถานการณ์ หรือแจ้งให้ผู้ที่เกี่ยวข้องทราบได้อย่างทันท่วงที เพื่อลดความเสียหายของสินค้า 
    \end{itemize}
    \item การติดตามสถานะและการแสดงผล
    \begin{itemize}
        \item ในระหว่างกระบวนการขนส่ง ระบบจะแสดงผลเฉพาะค่าอุณหภูมิและสถานะปัจจุบันแบบเรียลไทม์บนเว็บแอปพลิเคชัน เพื่อมุ่งเน้นให้ผู้ใช้งานสามารถเฝ้าระวังอุณหภูมิและรับมือกับความผิดปกติได้อย่างทันท่วงที 
        \item เมื่อกระบวนการขนส่งเสร็จสมบูรณ์ ระบบจะทำการดึงข้อมูลอุณหภูมิที่ถูกบันทึกไว้ตลอดการเดินทางมาประมวลผล เพื่อแสดงผลในรูปแบบกราฟเชิงเวลา โดยกราฟจะแสดงให้เห็นถึงการเปลี่ยนแปลงของอุณหภูมิตั้งแต่จุดเริ่มต้นการจัดส่งจนถึงจุดหมายปลายทาง พร้อมทั้งสรุปค่าทางสถิติเพื่อให้ผู้ใช้งานสามารถวิเคราะห์ภาพรวมของการขนส่งในรอบนั้น ๆ ได้
    \end{itemize}
    \item การปิดกระบวนการขนส่ง 
    \begin{itemize}
        \item เมื่อสินค้าถูกจัดส่งถึงปลายทางและผู้รับได้ทำการเซ็นยืนยันการรับสินค้าผ่านระบบเรียบร้อยแล้ว กระบวนการขนส่งจะถือว่าสิ้นสุดอย่างสมบูรณ์ และระบบจะบันทึกสถานะการส่งมอบเป็นขั้นตอนสุดท้าย 
        \item หลังจากที่สถานะการขนส่งสิ้นสุดลง ระบบจึงจะเปิดสิทธิ์ให้ผู้ใช้งานสามารถเข้าถึงและตรวจสอบประวัติข้อมูลอุณหภูมิย้อนหลัง ได้ตลอดช่วงระยะเวลาการเดินทางของสินค้า 
        \item ในขั้นตอนการตรวจสอบ ระบบจะทำการดึงข้อมูลอุณหภูมิทั้งหมดตั้งแต่เริ่มต้นจนถึงจุดหมายปลายทาง มาเปรียบเทียบกับข้อมูลที่ถูกบันทึกไว้ในระบบบล็อกเชน โดยอ้างอิงจากเวลาของแต่ละชุดข้อมูล
        \item กระบวนการนี้ทำเพื่อยืนยันความถูกต้องของข้อมูล และเป็นหลักฐานพิสูจน์เชิงประจักษ์ว่า ข้อมูลอุณหภูมิระหว่างการขนส่งไม่เคยถูกดัดแปลงหรือแก้ไข ทำให้สามารถใช้อ้างอิงได้ในกรณีที่เกิดปัญหาหรือข้อพิพาทเกี่ยวกับคุณภาพสินค้า
    \end{itemize}
\end{enumerate}
